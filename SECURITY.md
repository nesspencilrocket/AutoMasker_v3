# Security Policy

## このソフトウェアの想定範囲

AutoMasker は **ローカルの開発者ワークフロー** を対象としたツールです.
公開サーバや信頼できないクライアントからのアクセスを前提とした
プロダクション用途は想定していません.

## 既知のリスクと緩和策

### 1. HTTP 推論サーバ (`automasker.server`)

- **デフォルトで 127.0.0.1 (loopback) のみにバインド** します.
- ネットワーク公開するには `--allow-remote` + `--token <値>` の両方が必要です.
- すべてのエンドポイントは **Bearer token 認証** を要求します (無効化は loopback のみ).
- リクエストボディ上限 25 MB, 画像ピクセル数上限 5×10⁷ などで DoS を緩和しています.
- **HTTPS 終端は含まれていません**. 公開する場合は必ず nginx / caddy 等で
  TLS を被せてください.

### 2. モデルチェックポイントの信頼性

- `torch.load(..., weights_only=True)` を使用して pickle 由来の RCE を防いでいます.
- `scripts/download_checkpoints.py` は **HTTPS URL のみ許可** します.
- **SHA-256 検証について重要な注意**:
  本リポジトリ初期状態では期待ハッシュ (`KNOWN_HASHES`) が未登録 (`None`) です.
  初回ダウンロード時、スクリプトは計算された SHA-256 を表示して終了しますが、
  **検証は実施されません**. 検証を有効化するには:
  1. 初回ダウンロードで表示された SHA-256 を、公式配布元のチェックサム
     (もし公開されていれば) と手動で照合する
  2. 確認できた値を `scripts/download_checkpoints.py` の `FILES` 辞書に書き戻す
  3. 以降の実行は自動的に検証される (ファイル改ざん/破損を検出できる)

  このポリシーは、検証すべき正規ハッシュ値が上流公式から安定的に公開されて
  いない現状 (SAM2, Grounding DINO とも公式チェックサム文書なし) を反映しています.
- **信頼できないソースからダウンロードした `.pt` / `.pth` は使わないでください**.

### 3. 入力ファイル (画像 / 動画 / PLY)

- OpenCV, plyfile といったサードパーティライブラリに依存しているため、
  各ライブラリの脆弱性 (CVE) の影響を受けます. 定期的に依存を更新してください.
- 信頼できない `.ply` ファイルはロードしないでください (DoS 対策で 4 GB 上限は
  設定されていますが、それ以下での異常挙動は防げません).

### 4. 外部エンドポイント

- `scripts/download_checkpoints.py` 以外に外部通信するコードはありません.
- プロンプトやマスクが外部に送信されることはありません (完全ローカル動作).

## 脆弱性の報告

セキュリティ上の問題を見つけた場合は、**public issue を立てずに**
GitHub Security Advisory または管理者へ直接連絡してください.

## 推奨される使い方

- `automasker.server` は **ローカル開発専用** として扱う
- モデル重みは **公式サイトから直接ダウンロード** し、SHA-256 を確認する
- 本ツールの出力 `.ply` / マスクを他者に共有する際は、機密情報が
  写り込んでいないか確認する
